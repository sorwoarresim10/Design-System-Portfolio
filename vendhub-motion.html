01 · Problems & Philosophy

Motion Problems Found in the Codebase

The audit revealed 4 motion-related issues across the VendHub codebase. These stem from a lack of documented motion tokens and patterns.

🔴 P1 — animate-fade-in is broken
animate-fade-in is defined in tailwind.config.ts but references a fadeIn keyframe that doesn't exist in globals.css. The animation silently does nothing. Fix: add the keyframe definition.
🟡 P2 — No transitions on interactive controls
Checkbox, RadioGroup, and Select trigger have no transition-* classes. State changes (hover, checked) are instant instead of animated, creating a jarring feel.
🟡 P2 — Missing active:press feedback
Buttons have transition-colors but no active: press feedback (scale or opacity). Users get no tactile confirmation when clicking.
🟡 P2 — No motion governance
No documented easing curves, duration scale, or animation patterns. Each developer invents their own transition timing, leading to inconsistent feel across features.
Root cause: No motion token system existed. The fix requires (1) defining canonical easing and duration tokens, (2) creating named animation patterns, and (3) adding transition-* classes to all interactive components.
Philosophy
VendHub is a B2B ordering platform — motion must communicate state, not entertain. Every transition earns its place by making the interface feel responsive and predictable. Speed matters: operators reorder weekly, so animations that slow them down will get noticed and resented.
The Three Rules
1. Purposeful: Motion communicates state changes (hover, active, enter, exit) — never decoration.
2. Proportional: Duration matches element size. Small elements animate fast (75–150ms), large elements slower (300–500ms).
3. Accessible: All non-essential motion respects prefers-reduced-motion. Essential state changes use opacity, not transform.
02 · Easing

Easing Curves — 4 Tokens

Four standard curves cover every interaction. Hover over each card to preview the motion.

Ease Out
--ease-out
cubic-bezier(0, 0, 0.2, 1)
Fast start, decelerates. Use for elements entering the screen or expanding.
Ease In
--ease-in
cubic-bezier(0.4, 0, 1, 1)
Slow start, accelerates. Use for elements leaving the screen or collapsing.
Ease In-Out
--ease-in-out
cubic-bezier(0.4, 0, 0.2, 1)
Symmetrical. Use for elements repositioning within the layout.
Spring
--ease-spring
cubic-bezier(0.34, 1.56, 0.64, 1)
Overshoots slightly. Use sparingly for confirmations or delightful moments.
/* globals.css — Easing tokens */ :root { --ease-out: cubic-bezier(0, 0, 0.2, 1); /* Enter / expand */ --ease-in: cubic-bezier(0.4, 0, 1, 1); /* Exit / collapse */ --ease-in-out: cubic-bezier(0.4, 0, 0.2, 1); /* Reposition */ --ease-spring: cubic-bezier(0.34, 1.56, 0.64, 1); /* Delight — use sparingly */ }
When to use which: ease-out for ~80% of interactions (entering elements, hover). ease-in only for exits. ease-in-out for layout shifts (accordion, sidebar). spring only for success states (checkmarks, toasts).

06 · VendHub Patterns

VendHub-Specific Motion Patterns

Motion patterns specific to VendHub's product flows — ordering, navigation, and data-heavy interfaces. Based on audit of the Customer Experience screens.

Product Card Grid — Market Browse
Card hover: transition-shadow duration-150 with shadow elevation L1 → L2 (card-hover).
Add to cart: Button press active:scale-[0.98] + success icon swap with animate-fade-in.
Quantity stepper: transition-colors duration-100 on +/- buttons. Number change is instant (no animation on the number itself).
Checkout Flow — Multi-Step
Step transition: Content animate-fade-in (200ms) as each step loads. No slide — steps are not spatial.
Delivery method toggle: transition-all duration-150 for radio card selection border + background.
Order summary: Sticky panel — no motion on scroll. Line items are static.
Admin Dashboard — Data Tables
Table row hover: transition-colors duration-100 — fast, almost instant. Operators scan quickly.
Column sort: Instant reorder — no animation on data rows. Sort icon rotates with transition-transform duration-150.
Filter panel: animate-slide-down (200ms) from filter bar. Close with animate-fade-out (150ms).
Sidebar Navigation
Expand/collapse: transition-[width] duration-300 ease-in-out for sidebar width change (256px ↔ 64px).
Nav item hover: transition-colors duration-100 — background highlight.
Active indicator: Instant — no sliding indicator animation. Active state is a static background highlight.
VendHub motion principle: Data-heavy screens (tables, grids, dashboards) use faster durations (75–150ms). Content-focused screens (checkout, onboarding, settings) use standard durations (200–300ms). This keeps the interface snappy where speed matters and polished where comprehension matters.

08 · Accessibility

Reduced Motion & Accessibility

Users who enable prefers-reduced-motion in their OS settings must experience a fully functional interface without motion that could cause discomfort.

What to disable
Transform-based animations: translateY, scale, rotate — strip these entirely.
Decorative transitions: Card hover shadows, background color fades.
Shimmer/skeleton: Replace with static placeholder (no sweep).
Spring easing: Overshoot can cause nausea — remove.
What to keep
Opacity changes: Fade in/out still works — it's the gentlest transition.
Color changes: Instant color swap for focus, hover, active states.
Layout changes: Content must still appear/disappear — just instantly.
Loading spinners: Keep rotating — users need to know something is loading.
/* Pattern: Wrap non-essential animations */ @media (prefers-reduced-motion: no-preference) { .animate-on-enter { animation: fadeIn 200ms var(--ease-out) both; } } /* Pattern: In React components */ const prefersReducedMotion = window.matchMedia('(prefers-reduced-motion: reduce)').matches; const duration = prefersReducedMotion ? 0 : 200;
WCAG 2.1 Level AA requirement: Any animation that starts automatically, lasts longer than 5 seconds, and is presented alongside other content must have a mechanism to pause, stop, or hide it. The shimmer loader satisfies this — it stops when content loads.
09 · Rules

Do's & Don'ts

The most common motion mistakes and their corrections.

✓ Do
Use transition-colors duration-150 for all hover color changes. This is the default transition for every interactive element.
✗ Don't
Don't animate more than 3 properties simultaneously. Each additional property adds visual noise and reduces GPU performance.
✓ Do
Use ease-out for entering elements, ease-in for exiting. This matches natural physics — objects accelerate into view and decelerate out.
✗ Don't
Don't use durations above 500ms except for full-page transitions. Anything slower feels laggy in a B2B ordering app.
✓ Do
Respect prefers-reduced-motion — wrap all non-essential animations in @media (prefers-reduced-motion: no-preference).
✗ Don't
Don't use linear easing for UI interactions. It feels mechanical and unnatural. Only use linear for progress bars and shimmer loaders.
✓ Do
Match duration to element size: small = fast, large = slow. Button press (75ms), dropdown (200ms), modal (300ms), drawer (400ms).
✗ Don't
Don't use ease-spring on form controls or navigation. Spring is for delight moments (success checkmark, celebration) — not utility interactions.
✓ Do
Use CSS transition for state changes (hover, focus, active). Use CSS animation for enter/exit (elements appearing or disappearing).
✗ Don't
Don't animate layout properties (width, height, top, left). These trigger layout recalculation. Use transform and opacity instead — they're GPU-accelerated.
Cross-reference: This motion system is designed to work with the VendHub Shadow/Elevation system (L1–L5) for hover effects, the Border Radius system for rounded transitions, and the Opacity system for disabled/overlay states. All tokens are defined in the same globals.css :root block.

05 · Why this matters

Without motion system vs. with motion system

Without defined tokens, developers invent their own timing. The same dropdown opens differently across 3 features.

✕ Without motion tokens
🔴 animate-fade-in references non-existent keyframe (broken)
🔴 Checkbox/Radio/Select have no transitions (state changes are instant)
🔴 Buttons have no active:press feedback
🟡 Dropdown A opens in 150ms, Dropdown B opens in 300ms
🟡 Modal uses linear easing (feels mechanical)
🟡 No prefers-reduced-motion support
🟡 Each developer picks their own duration
✓ With VendHub motion tokens
✅ fadeIn keyframe defined — animate-fade-in works
✅ All controls: transition-colors duration-150
✅ active:scale-[0.98] active:opacity-90 on every button
✅ All dropdowns: animate-slide-up (200ms, ease-out)
✅ All modals: animate-scale-in (300ms, ease-out)
✅ @media prefers-reduced-motion respected
✅ 7 named duration tokens — no guessing
