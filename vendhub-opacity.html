01 · Philosophy

Why Semantic Opacity Tokens

Without a system, developers write opacity-50 in one file, opacity-40 in another, and bg-black/60 in a third — all meaning "disabled." Named tokens eliminate this drift.

Intent, Not Numbers
opacity-38 means "disabled" across the entire app. The number 38 comes from Material Design 3's disabled standard (0.38). Every developer uses the same value without guessing.
Two Application Methods
Element opacity: disabled:opacity-38 — dims the entire element. Color modifier: bg-primary/16 — applies opacity to a single color channel. Choose based on context.
Contrast-Safe
Every opacity stop has been tested against VendHub's color palette for contrast compliance. Low opacity values (4%, 8%, 16%) are only for backgrounds, never for text.

Why 38% for disabled? Material Design 3 uses 0.38 opacity for disabled content. At this level, the element is clearly non-interactive while still being readable enough that users can identify what the disabled control represents. VendHub adopts this standard for cross-platform consistency.

State-Specific Usage Map

How each opacity token maps to specific component states. This is the definitive reference for which token to use when.

StateTokenHow to ApplyTailwind Example

Hover background --opacity-16 Apply to a color as background modifier hover:bg-primary/16 or hover:bg-neutral-900/16
Focus ring bg --opacity-8 Optional tinted focus halo behind the ring focus-visible:bg-primary/8

AlertDialog overlay --opacity-50 Match Dialog exactly bg-black/50 backdrop-blur-sm

Tinted surface --opacity-4 Subtle tinted backgrounds (banners) bg-primary/4 or bg-error/4
Disabled state rule: Always apply opacity-38 to the element itself, never to a parent container that holds focusable children. If a parent is dimmed, screen readers may still focus children inside it, creating a confusing experience.
05 · Tailwind Patterns

Copy-Paste Component Patterns

The most common opacity patterns in VendHub components. Use these exactly.

Disabled Button
A disabled button dims the entire element — background, text, and icon all at 38% opacity. The cursor changes to not-allowed.
/* Disabled button */ className="bg-primary-500 text-white disabled:opacity-38 disabled:cursor-not-allowed"
Hover Background Tint
Instead of a separate hover color, use the base color with 16% opacity. This works with any color — primary, neutral, error — without defining new color stops.
/* Ghost button hover */ className="hover:bg-primary/16 transition-colors duration-150" /* Table row hover */ className="hover:bg-neutral-900/8" /* Nav item hover */ className="hover:bg-neutral-500/16"
Modal Overlay
All modal backdrops use the same opacity + blur combination. This ensures consistent dimming across Dialog, AlertDialog, and Sheet.
/* Dialog overlay */ className="fixed inset-0 bg-black/50 backdrop-blur-sm z-overlay"
Status Banner Background
Success, error, warning, and info banners use 4% opacity of their semantic color on a white surface. Subtle enough to not overwhelm, strong enough to signal state.
/* Success banner */ className="bg-success-500/4 border border-success-200" /* Error banner */ className="bg-error-500/4 border border-error-200"
Active / Pressed State — Full Pattern
The press state combines opacity-90 with a subtle scale-down. This gives tactile feedback without jarring visual changes. Used on all buttons and clickable cards.
/* Button with full interaction states */ className="bg-primary-500 text-white hover:bg-primary-600 active:opacity-90 active:scale-[0.98] focus-visible:ring-1 focus-visible:ring-primary-500 focus-visible:bg-primary/8 disabled:opacity-38 disabled:cursor-not-allowed transition-colors duration-150"
06 · Rules

Do's & Don'ts

✓ Do
Use opacity-38 consistently for all disabled controls — buttons, inputs, selects, checkboxes, radio buttons. One value across the entire app.
✗ Don't
Don't use opacity-50 on text — it makes it harder to read. For de-emphasized text, use a lighter color token (text-neutral-500) instead of reducing opacity.
✓ Do
Use Tailwind's color modifier syntax bg-primary/16 for hover states. No separate class needed — the /16 applies 16% opacity to that specific color.
✗ Don't
Don't use arbitrary opacity values like opacity-[0.23] or bg-black/[0.35]. If a value isn't in the 9-token scale, round to the nearest stop.
✓ Do
Use opacity-50 for all modal backdrops system-wide. Dialog, AlertDialog, Sheet, and any future overlay must use the same scrim value.
✗ Don't
Don't apply opacity-38 to a parent that contains focusable children. Apply it to the element itself. A dimmed parent can still receive focus on its children, confusing screen reader users.
✓ Do
Prefer opacity modifiers over hardcoded semi-transparent hex values. bg-primary/16 is readable and maintainable; bg-[rgba(34,197,94,0.16)] is not.
✗ Don't
Don't use opacity on brand colors for overlays — use bg-black/50 for scrims, not bg-primary/50. Green-tinted overlays look wrong and reduce content readability.
07 · Config

Complete Tailwind Configuration

Copy-paste into globals.css and tailwind.config.ts.

/* ── globals.css — Add to :root ─────────────────────────── */ :root { --opacity-100: 1; --opacity-90: 0.9; --opacity-75: 0.75; --opacity-60: 0.6; --opacity-50: 0.5; --opacity-38: 0.38; --opacity-16: 0.16; --opacity-8: 0.08; --opacity-4: 0.04; }
/* ── tailwind.config.ts — theme.extend.opacity ─────────── */ opacity: { '0': '0', '4': 'var(--opacity-4)', '8': 'var(--opacity-8)', '16': 'var(--opacity-16)', '38': 'var(--opacity-38)', '50': 'var(--opacity-50)', '60': 'var(--opacity-60)', '75': 'var(--opacity-75)', '90': 'var(--opacity-90)', '100': '1', }
/* ── Usage cheat sheet ──────────────────────────────────── */ /* Element opacity — dims everything */ className="disabled:opacity-38" /* disabled */ className="active:opacity-90" /* pressed */ className="placeholder:opacity-75" /* placeholder */ /* Color modifier — opacity on a single color */ className="bg-black/50" /* overlay scrim */ className="hover:bg-primary/16" /* hover tint */ className="focus-visible:bg-primary/8" /* focus halo */ className="bg-success-500/4" /* success banner */ className="bg-error-500/4" /* error banner */
Cross-reference: This opacity system works with the Z-Index system — modal overlays use bg-black/50 at z-overlay (30). It also works with the Motion system — disabled elements transition to opacity-38 using transition-opacity duration-150. And with the Color system — all color modifier values (/4, /8, /16) use the semantic color tokens.
